Substrate EVM tracing.

The purpose of this extension is enable tracing the EVM opcode execution and will be used by both Dapp developers - to get a granular view on their transactions - and indexers to access the EVM callstack (internal transactions).

The extension composed of two modules:

  • Runner - interfaces the runtime Api with the EVM Executor wrapper.
  • Executor Wrapper - an evm::Handler implementor that wraps an evm::StackExecutor.

The wrapper replaces the original recursive opcode execution done by the evm's create_inner and call_inner methods, with one that allows capturing the intermediate machine state between opcode executions (stepping), resulting in either a granular per opcode response:
